# Marketing Outreach Ops

Use this skill to run a tight execution loop for marketing and outbound operations.

## When to Use

- User wants to build pipeline or outbound campaigns.
- User asks for competitor or market research.
- User asks for personalized outreach drafts.
- User asks to automate recurring research or reporting.
- User wants ongoing improvement from feedback and outcomes.

## Core Workflow

1. Clarify objective
- Capture one primary KPI (for example: booked calls, reply rate, SQL conversion).
- Capture ICP and constraints (industry, role, geography, tone, channels).

2. Build task plan with `todo`
- Create a short step list before broad execution.
- Keep at most one active workstream at a time unless asked for parallel runs.

3. Research with tools
- Use `web_search` for source discovery.
- Use `web_extract` for evidence gathering and synthesis.
- Record source-backed claims and explicit assumptions.

4. Produce deliverables
- ICP snapshot
- Messaging angles (problem-led, outcome-led, proof-led)
- Outreach sequence drafts (email/DM/follow-up variants)
- Objection handling snippets
- CTA options by funnel stage

5. Persist learnings
- Save durable findings to memory:
  - audience pain points
  - performance patterns
  - user preferences
  - winning hooks and failed hooks

6. Improvement loop
- Request quick feedback after each batch output:
  - score 0-1
  - what worked
  - what missed
- Adjust future strategy based on this feedback.

## Output Standards

- Use concrete language and avoid fluffy claims.
- Separate facts vs assumptions.
- Include one recommended next action at the end.
- For outreach copy:
  - keep first touch concise
  - include role-specific context
  - include exactly one clear CTA

## Safety and quality checks

- Do not fabricate metrics or references.
- Flag uncertainty when source quality is weak.
- If prospect/company specifics are missing, ask for only the minimum missing data.
